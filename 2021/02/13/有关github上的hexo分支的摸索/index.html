有关github上的hexo分支的摸索

2021-02-13- 2021-02-14

问题探索

今天一直在对博客上的“文章发表”进行实验，但是无法在github上查看hexo的源代码，我无法从github上跟其他博客的代码进行比对和改良。后来问过云游君后，大佬告诉我，我的git页面代码应该有两个分支，一个是hexo的源代码分支，另一个是用于存放博客的内容分支，随后我去查看我的分支情况（因为之前我比对代码的时候只剩下内容分支，导致我心理很恐慌），我发现的我的源代码分支不知道什么时候不见了，可能是年前误操作或者是没有备份，导致那一分支消失，现在只剩下名为main的内容分支。

根据云游君的建议，我的博客确实是部署到了github上，我也可以通过sublime-text来对文章内容进行修改，即使不备份hexo的源文件我也可以正常进行内容更新，但比对代码成了大麻烦（我只是个小菜鸟QAQ）。所以我还是需要对hexo进行备份，并添加分支到git上，于是我找了类似的博客如EnjoyToShare,并希望问题得以解决。

本地克隆

经过以上努力，我又进行了几个小时的探索通过Hua’s Cabin的博客，我最终找到了答案。基本思路就是将刚才新创立的放置hexo的源代码的分支（此时与main相同，可视作空分支）通过指令克隆到本地，再将本地的hexo源代码统统放入这个分支文件夹中，然后剔除主题下的.git目录防止两个分支主题冲突，接着修改.gitnote文件（用于剔除上传文件中不属于源代码的冗余文件），这就完成了这个分支的全部，之后就可以通过相关的git命令将本地的hexo源代码文件全体上传至远程仓库中的hexo分支中。

最后，完成以上任务，就可以实现同一个仓库下的一个分支存储hexo源码，另一个分支分存储静态网页和博客，二者秋毫无犯（！完美！）
